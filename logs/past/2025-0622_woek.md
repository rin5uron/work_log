# 【2025/05/25】フールプルーフ演習①のレスポンシブデザイン修正

## 完了したこと ✅
- フールプルーフ演習①のレスポンシブデザイン問題を解決
- 画像が枠に被る問題を修正（margin-top: 40px追加）
- スマホ表示時の各要素間の余白を調整
- 枠の高さを統一して文字サイズのバラつきを解消（min-height設定）
- 重複したCSSコードの整理とリファクタリング

## 学んだこと 📚

### CSSレスポンシブデザインのトラブルシューティング：
- **メディアクエリが効かない原因**：
  - `body`に`display: flex`と`height: 100vh`が設定されていると、ページ全体が横並びレイアウトになってしまう
  - 既に縦並びの要素に`flex-direction: column`を指定しても変化が見えない
  - ブレークポイント（768px）が狭すぎてタブレットサイズで適用されない場合がある

- **効果的な修正方法**：
  - `body`の不要な`display: flex`を削除して通常の縦並びレイアウトに変更
  - メディアクエリのブレークポイントを1024pxに拡張（タブレットも含む）
  - 各要素に適切な`margin`と`gap`を設定して視覚的な余白を確保
  - `min-height`を使用して枠の高さを統一し、文字サイズの見た目を揃える

- **CSSの整理とメンテナンス**：
  - 重複したCSSセレクタは後から読み込まれたものが優先される
  - 同じクラス名の定義が複数あると予期しない表示になる可能性
  - 定期的なコードレビューとリファクタリングが重要

### 今日学んだ用語 📖
- **メディアクエリ（Media Query）**：画面サイズに応じてCSSを切り替える仕組み
- **ブレークポイント（Breakpoint）**：レスポンシブデザインで表示を切り替える画面幅の境界値
- **ビューポート（Viewport）**：ブラウザでWebページが表示される領域
- **リファクタリング（Refactoring）**：機能を変えずにコードの構造を改善すること


## メモ 📌
- レスポンシブデザインは最初から考慮して作ると後から修正が楽
- 成果を誰かに共有することで、レスポンシブデザインを考えたりちょっとした工夫をしようとする意欲が生まれた

# 【2025/05/15】.gitignoreファイルの重要性について学習

### 完了したこと ✅
- .gitignoreファイルの役割と配置場所について理解
- .gitignoreファイルの復旧と正しい設置

### 学んだこと 📚
- .gitignoreファイルの重要性
  - Gitが追跡しないファイルを指定するための設定ファイル
  - node_modules、.DS_Storeなどの不要ファイルをGit管理から除外できる
- .gitignoreはプロジェクトのルートディレクトリに配置する必要がある
  - gitはルートディレクトリから順に階層的に管理されるため、上層階に置くことでエラー防止にもなる
  - 別の場所に移動すると、今まで無視されていたファイルが追跡対象になってしまう
- .gitignoreファイルの移動による影響
  - アンラック（untracked）状態のファイルが大量に表示される
  - 特にnode_modulesフォルダやシステムファイル（.DS_Store）が影響を受ける
- 対処法
  - .gitignoreファイルをプロジェクトルートに戻す
  - 正しい無視設定を記述する

### メモ 📌
- リポジトリ階層を深くしすぎるとエラーの原因になる

  - コスト：コーディングは基本無料（サーバー代のみ） vs ノーコードは無料プランあり、拡張すると有料

- documentオブジェクトの使い方（超簡単に説明）
  - documentは「今見ているページの設計図」のようなもの
  - リモコン（JavaScript）でテレビ（HTML）を操作するための橋渡し役
  - 例えば：
    ```javascript
    // HTMLの「id=counter」という名札がついた部分を探す
    const 数字表示部分 = document.getElementById("counter");
    
    // その部分の表示内容を「5」に変える

    ```
- IDは「探し物をするための名札」のようなもの

- メソッドチェーン（連鎖）について
  - 複数のメソッドを `.` でつなげて一気に書く方法
  - 例： `document.getElementById("plus").addEventListener("click", function() {});`
  - 一時変数を作らずに済むので短く書ける
  - 「料理の材料を用意して→切って→炒めて→盛り付ける」という一連の流れを一息で表現する感じ
  - 詳しいメモは `memo_study.md` に記録

### 発生したエラーと解決策 🐛
- **問題点**: `addEventListener`が作用しない
- **原因**: イベントリスナーを`document`に設定していたが、クリックされるのはプラスボタン
- **解決策**: `document.addEventListener`ではなく、プラスボタンに直接イベントリスナーを設定
  ```javascript
  // 誤り
  document.addEventListener("click", function() {
    counter++;
  });
  
  // 正解
  incrementBtn.addEventListener("click", function() {
    counter++;
  });
  ```
- **学び**: イベントリスナーは「どの要素の」「どんなイベントを」待ち受けるかを明確にする必要がある

### 【2025/04/20】JavaScript基礎学習：配列とループ処理

#### 完了したこと ✅
- forEachメソッドの基本的な使い方を学習
- 配列のインデックスについて理解
- 実際のコードで動作確認

#### 学んだこと 📚
##### 配列のインデックス
- 配列の中の位置を表す番号（0から始まる）
- 例：`month_list = ["1月","2月","3月"]`の場合
  - `month_list[0]` → "1月"
  - `month_list[1]` → "2月"
  - `month_list[2]` → "3月"
- インデックスは必ず0から始まることに注意

##### forEachメソッド
- 配列の要素を1つずつ順番に処理するための命令
- アロー関数と組み合わせて使用
- 基本形：`配列.forEach(要素 => { 処理 })`
- 例：
  ```javascript
  month_list.forEach(month => {
      console.log(month);
  });
  ```
- forループで書く場合との比較：
  ```javascript
  // forループの場合
  for (let i = 0; i < month_list.length; i++) {
      console.log(month_list[i]);
  }
  
  // forEachの場合
  month_list.forEach(month => {
      console.log(month);
  });
  ```
- forEachを使うメリット：
  - インデックスを意識せずに書ける
  - コードがシンプルになる
  - 配列の長さを気にしなくて良い

#### メモ 📌
- 配列は0から数え始めることを常に意識する
- forEachは配列専用のメソッドで、配列の操作がシンプルに書ける
- 従来のforループも状況によって使い分ける

  この組み合わせで、子要素を横方向に両端揃え、縦方向に中央揃えにできる

- displayプロパティの多様な値と使い分け:
  - block, inline, inline-block, none, flex, gridなど
  - それぞれの表示特性と適切な使用場面

- align-itemsプロパティの値と効果:
  - stretch（デフォルト）: 子要素を親の高さに合わせて伸縮
  - flex-start: 上揃え
  - flex-end: 下揃え
  - center: 中央揃え
  - baseline: テキストのベースラインで揃える

- 重複するCSSコードを整理することで、ファイルサイズを削減し可読性を上げられる
- 全角文字（スペース、コロン）がCSSファイルに入るとエラーの原因になる

### 今日学んだ用語 📖
- `display: flex` - 要素をフレックスコンテナに変換するプロパティ
- `justify-content: space-between` - 子要素を両端揃えにするプロパティ
- `align-items: center` - 子要素を縦方向中央に配置するプロパティ
- `display: grid` - 要素をグリッドコンテナに変換するプロパティ
- `align-items: baseline` - 子要素をテキストのベースラインで揃えるプロパティ
- `font-weight` - テキストの太さを指定するプロパティ（normal=400、bold=700）
- `em` - 親要素のフォントサイズを基準とする相対的な単位
